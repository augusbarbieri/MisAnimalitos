Manual de Arquitectura MVC: Proyecto Manadas

Este documento es tu guía de estudio definitiva para defender tu proyecto final. Aquí encontrarás cómo está estructurado el código, dónde está cada cosa y cómo responder a las preguntas más comunes de los profesores.

1. Concepto General: Arquitectura MVC

Tu proyecto está construido utilizando el Patrón de Diseño MVC (Modelo-Vista-Controlador). Esto significa que el código está separado en tres capas distintas para que sea ordenado, seguro y escalable.

  • M (Modelos): Son los únicos archivos que hablan con la Base de Datos (MySQL). Si hay un SELECT, INSERT o UPDATE, está aquí.
  • V (Vistas): Son los archivos HTML/PHP que el usuario ve en la pantalla. No tienen lógica compleja, solo dibujan tablas y formularios.
  • C (Controladores): Es el "cerebro". Recibe lo que el usuario hace (clics), le pide información al Modelo y decide qué Vista mostrar.

3. Flujos de Trabajo (Cómo funciona por dentro)

A. Flujo de Login (Autenticación)

  1. El usuario entra a views/usuario/login.php y pone su usuario/clave.
  2. Al presionar "Ingresar", el formulario se envía por POST a controllers/AuthController.php.
  3. El Controlador llama a AuthModel->autenticarUsuario().
  4. El Modelo busca en las 3 tablas. Si lo encuentra, devuelve un rol.
  5. El Controlador guarda el rol en $_SESSION y redirige al panel correcto.

B. Flujo del Dueño (Ejemplo: Pedir Paseo)

  1. El dueño entra a UsuarioController.php?action=solicitar_paseo.
  2. Elige perro y paseador, y envía el formulario.
  3. El Controlador llama a UsuarioModel->solicitarPaseo().
  4. IMPORTANTE: Ese Modelo usa una transacción en MySQL. Inserta en la tabla paseo y luego en pago. Si falla, cancela todo.

4. Preguntas Frecuentes de Examen (Cheat Sheet)

Pregunta 1: "¿Por qué dividiste el código en tantas carpetas?"
Respuesta: "Para aplicar el patrón MVC. Esto separa la lógica de base de datos (Modelos) del diseño visual (Vistas). Así, si mañana quiero cambiar los colores, no toco el código de la base de datos. Es el estándar de la industria."
Pregunta 2: "Muéstrame dónde te conectas a la Base de Datos"
Respuesta: "Está centralizado. Solo tienes que abrir config/conexion.php."
Pregunta 3: "¿Cómo evitas que un paseador entre al panel del administrador?"
Respuesta: "En la cabecera de cada Controlador, importo sesion.php y uso la función get_user_role(). Si el rol no coincide con el necesario, uso die() para bloquear el acceso."
Pregunta 4: "Muéstrame cómo traes los paseos para el administrador."
Respuesta: "Abro models/AdminModel.php en la función getPedidos(). Ahí tengo un SELECT con varios LEFT JOIN para unir las tablas de paseo, pago, mascota, usuarios y paseador."
Pregunta 5: "¿Cómo funciona el cierre de sesión?"
Respuesta: "Se envía la petición a AuthController.php?action=logout. Este llama a cerrarSesion() (que destruye las cookies usando session_destroy()) y redirige al login."
Pregunta 6: "¿Por qué en tu base de datos algunas variables están en sesión?"
Respuesta: "Usamos $_SESSION de PHP para mantener un estado ('state') en el servidor. Al loguearse, guardamos su user_id y rol. PHP almacena esto de forma segura en el servidor."
